# 情报学中文论文实验部分撰写技能

本技能用于根据用户提供的论文框架、方法描述和实验结果数据，撰写符合中文情报学核心期刊规范的实验部分。

## 总体原则

情报学中文论文的实验部分需要做到：严谨而不晦涩，详尽而不冗余，数据驱动而有深度分析。实验部分既是方法有效性的证据，也是读者理解研究贡献的关键窗口。

**核心要求**：实验部分必须以完整论文文段的形式呈现，使用连贯的自然段落展开论述，严禁使用分点列表、条目罗列或提纲形式。每一小节都应是若干完整段落组成的论述单元，段落之间逻辑连贯、论证严密。

## 实验部分的标准结构

一篇完整的情报学中文论文实验部分通常包含以下子章节，按顺序组织：

### 1. 数据集（必写）

这一节说明实验所使用的数据从何而来、如何处理、最终规模如何。需要包含：

- **数据来源**：明确说明数据的出处（如政府网站、学术数据库、社交平台等），并说明采集方式（爬虫工具、API接口、手工收集等）。如果使用了特定平台或工具，应给出名称。
- **数据类型与内容**：描述数据的具体类别（如政策正文、问答数据、评论文本等），并简要说明每类数据的特征（如客观性、权威性、多样性等）。
- **数据清洗过程**：列出具体的清洗步骤，如去重、格式统一、噪声去除、人工校验等。这部分要体现数据质量保障的严谨性。
- **最终数据规模**：用具体数字呈现，如"共获得XX条/份数据"。
- **特殊数据集的构建**：如果研究需要构建微调数据集、验证集等特殊数据集，需单独说明其构建方法、标注格式、规模和用途。可使用表格展示数据集示例。

**写作要点**：数据集描述需让读者能够理解数据的质量和代表性，必要时使用表格或图来辅助说明数据格式。

### 2. 评测指标（必写）

这一节定义用于衡量实验效果的指标。需要包含：

- **指标选择的理由**：简要说明为什么选择这些指标（如"本文重点关注XX与XX的融合，因此更注重对XX效果的评估"）。
- **每个指标的定义与公式**：
  - 先用自然语言说明指标衡量的是什么
  - 再给出数学公式（使用规范的数学符号）
  - 最后解释公式中每个符号的含义
- **值域与解释方向**：说明指标的取值范围（如0到1之间），以及高值或低值分别代表什么意义。

**写作要点**：即使是常见指标（如Precision、Recall、F1），在情报学论文中也要给出完整定义，因为读者可能跨学科。公式编号应连续。每个指标建议用独立的小标题或编号标记。

### 3. 实验设置与对比模型（必写）

这一节说明实验的具体配置。需要包含：

- **对比模型/基线方法**：列出所有对比的模型或方法，并简要说明每个模型的背景（如"ChatGPT-3.5作为通用大模型的基线"）。说明为什么选择这些模型作为对比对象。
- **参数设置**：以表格形式呈现关键参数（如学习率、batch size、温度参数、Top-K等），包括参数名称、含义和具体值。
- **实验环境**（如有必要）：硬件配置、软件版本等。

### 4. 实验结果与分析（核心，必写）

这是实验部分的核心，需要做到数据展示与深度分析并重。包含两个层面：

#### 4.1 定量指标分析

- **结果呈现**：使用表格展示各模型在各指标上的得分，表格标题使用中英双语（如"表X 平均指标得分 / Table X Metric Score"）。
- **横向对比**：比较不同模型在同一指标上的表现，给出具体的百分点差异（如"较XX模型提升了X.X个百分点"）。
- **纵向解读**：对每个指标逐一分析结果背后的原因：
  - 为什么某模型在某指标上表现更好？（可从训练数据、模型架构、参数设置等角度解释）
  - 为什么某个指标整体得分较低或较高？（与任务特点、数据特征关联）
  - 指标之间的关联与对比（如"答案相关性普遍低于事实一致性但高于上下文精确度"）
- **关键发现总结**：每段分析结束后归纳核心结论。

**分析深度的要求**：不要只说"A比B好"，要说"A比B好，这是因为……，这说明……"。好的分析有三个层次：
1. 描述现象（结果是什么）
2. 解释原因（为什么会这样）
3. 引申意义（这说明了什么、未来可以怎么改进）

#### 4.2 输出结果分析（定性分析）

- **场景设计**：选择2-3个代表性的应用场景进行案例展示
- **对比展示**：以表格形式并排展示提出方法与基线方法的输出结果
- **定性评价**：从准确性、完整性、可理解性、实用性等维度进行文字评价
- **表格中的长文本**：输出对比表格中可以包含较长的文本内容，以便读者直观感受差异

### 5. 实验结果讨论（推荐写）

这一节对实验结果进行更高层次的讨论，通常组织为几个要点：

- **方法的优势总结**：概括提出方法为什么有效（从机制层面解释）
- **数据集/标注的影响**：讨论数据质量对实验结果的影响
- **局限性与改进方向**：坦诚指出当前方法的不足，并提出未来改进思路

每个讨论要点建议用"（X）标题"的格式编号。

## 写作风格规范

### 语言风格
- 使用规范的中文学术语言，句式正式但流畅
- 术语首次出现时给出英文全称和缩写，如"检索增强生成（Retrieval-Augmented Generation，RAG）"
- 数字表述：百分比用"%"、具体数值保留适当小数位
- 对比表述常用句式："较XX模型提高/提升了X.X个百分点"、"在XX指标上分别取得XX、XX和XX个百分点的提升"

### 表格规范
- 表题使用中英双语，格式为"表X 中文标题 / Table X English Title"
- 表格中数值对齐，百分比保留一位小数
- 最优结果可加粗标注
- 参数表应包含：参数名称、参数含义、参数值三列

### 图表规范
- 图题使用中英双语，格式为"图X 中文标题 / Fig.X English Title"
- 流程图、框架图中的元素标注应清晰

### 段落组织
- 每段围绕一个核心观点展开
- 段首通常是该段的主题句
- 段末通常是总结或过渡句
- 分析段落的典型结构：现象描述 → 原因分析 → 意义引申
- **关键原则**：所有正文必须使用完整段落撰写，严禁使用列表、分点或提纲形式替代段落论述。每一小节应由若干自然段落组成，段落之间逻辑连贯、过渡自然。

## 输入要求

使用此技能时，Claude需要向用户收集以下信息（如果尚未提供）：

1. **研究方法概述**：提出了什么方法/框架？核心创新点是什么？
2. **实验数据**：数据来源、规模、类型
3. **评测指标及结果**：使用了哪些指标？各模型的得分是多少？
4. **对比模型**：与哪些模型进行了对比？
5. **关键参数设置**：实验中的重要参数及其值
6. **代表性案例**（如有）：输出结果的具体示例

## 输出格式

生成的实验部分应为完整的中文学术论文章节，包含：
- 章节编号和标题（如"4 实验"）
- 子章节结构完整
- 表格以Markdown格式呈现（后续可转换为Word或LaTeX）
- 公式以LaTeX数学符号呈现
- 参考文献引用以"[数字]"格式标注

**重要强调**：所有正文内容必须以完整段落形式呈现（完整论文文段），即使用连贯的学术 prose 撰写，严禁使用分点列表、条目罗列或提纲形式。每一小节都应是若干自然段落组成的完整论述，符合正式期刊论文的写作规范。实验结果分析要有深度，通过完整句子阐述现象、解释原因、引申意义，而非简单罗列要点。

最终输出应保存为.md文件（便于用户查看和编辑），如需要也可使用docx skill转换为Word文档。

```
4 实验

    （2）XX数据集
    （3）验证数据集
  4.2 评测指标
    （1）指标A
    （2）指标B
    （3）指标C
  4.3 实验结果
    （1）指标评测结果分析
    （2）输出结果分析
  4.4 实验结果讨论
    （1）讨论要点一
    （2）讨论要点二
    （3）讨论要点三
```

注意：具体章节编号要与用户论文的整体结构保持一致（如果用户前面的章节已经用到第3章，则实验部分从第4章开始）。
