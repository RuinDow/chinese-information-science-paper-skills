# Paper Method Generator

本技能专用于撰写学术论文的**方法框架部分**（Methodology / Method Section）。根据整篇论文的结构和内容要求，自动生成连贯、完整、专业的方法论文段（约2500字），以流畅的学术 prose 形式详细阐述研究设计与技术实现，不采用列点或清单形式呈现。


## 工作流程与撰写要求

在接到用户的论文结构或内容要求时，必须严格按照以下**分步流程**生成方法框架部分，确保最终结果是**完整论文文段**。

### 第一步：画出示意图（使用 Mermaid）
- 在正式撰写文段之前，**必须首先画出研究框架示意图**（通常为流程图或架构图）。
- 示意图必须使用 `mermaid` 语法进行绘制。
- 必须使用标准的 Markdown 代码块包裹图表代码（如：````markdown ```mermaid [代码内容] ``` ````）。
- 示意图应当独立存在，将其放置在方法部分的开头，通过代码块与后续的撰写文段完全分离开来。

### 第二步：撰写完整论文文段
- 在框架图完成之后，紧接着开始文字部分的撰写。
- **极度重要：必须生成“完整论文文段”！** 所有的内容（功能定位、技术实现、公式推导、参数选择等）必须完全融入连贯流畅的、大段落的学术论述中。
- **严禁列点式回答**：绝对禁止使用任何形式的列点、清单或条目化格式呈现（不要出现 `-`, `*`, `1. 2. 3.` 等）。
- 篇幅需控制在 **2500-3000字**。确保这些内容言之有物，避免空泛的辞藻堆砌，技术细节必须与整个论文的研究主题高度相关。
- 最终输出必须为标准的 Markdown 格式，且方法部分以段落式的 prose 呈现。

### 第三步：写完后进行检查（Self-Check）
- 在输出全部内容之后，**必须自我进行一次强制检查环节**，审视所生成的内容。
- **检查内容 1**：是否画了 Mermaid 示意图且和正文部分分开了？
- **检查内容 2**：是否是**完整论文文段**？全文（特别是核心模块设计中）有没有任何隐蔽的列点或列表结构？一旦发现自己写了列表，必须立即组织成完整的长句和长段落重新输出。

## 核心逻辑与写作要点

这是论文方法框架部分的核心，必须以**连贯的学术文段**清晰展示"你做了什么、怎么做的"。

**标准组织方式：**

方法框架部分的结构通常遵循"总-分"逻辑，建议采用如下层次组织内容：首先是研究框架或整体架构的概述，以宏观视角呈现研究设计的全貌；随后依次展开各功能模块的详细设计说明，对每个模块进行深入阐释。这种组织方式使读者能够先建立整体认知，再逐步深入了解各组成部分的技术细节。

**写作要点：**

**(1) 先总后分：** 方法框架的开篇应首先以完整段落配合研究框架图概述整体研究流程或系统架构，为读者提供全局性认识。在概述整体流程时，可采用"整体研究框架如图1所示"或"本文提出的XX框架流程如图2所示，其运作机制可概括为A-B-C三步骤"等句式自然引出框架图，并配合文字对核心流程进行简要说明。

**(2) 框架图的分离处理：** 再次强调，第一步画出的研究框架图（mermaid 代码块）应放置在方法开头并与完整论文文段完全分离开来。在此之后，全文仅留下文字论述作为支撑。

**(3) 模块描述的完整文段要求：** 对各功能模块的描述必须以连贯的学术 prose 形式呈现，避免使用列点或清单。每个模块的阐述应自然融入段落之中，依次说明该模块的功能定位（即该模块承担什么任务、解决什么问题）、具体的技术实现方法（即采用了什么算法、模型或技术路线，以及具体的实现细节），以及关键参数设置或设计选择的理由（即为何如此设计，其理论依据或实证考量是什么）。这三方面内容应在段落中层层递进、有机融合，形成逻辑严密的完整论述。

**(4) 公式的学术化融入：** 若涉及数学公式，应将其自然嵌入段落文字之中。具体而言，公式需标注编号（如"如公式(1)所示"），公式之后应立即跟随变量说明（如"其中，F表示事实一致性得分；Ca表示……"），确保公式前后均有充分的文字铺垫和解释，使公式成为论述逻辑的有机组成部分，而非孤立存在的数学表达。

### 撰写风格要求
方法框架部分的撰写必须输出**完整论文文段**。优先采用结构完整、信息密度高的长句，通过从句、并列结构、递进关系等方式将多层含义有机融合在连贯的句子中表达，使行文紧凑流畅，充分体现学术写作的严谨性与专业性，避免频繁使用短句导致文意碎片化。同时，句与句之间、段与段之间的衔接必须自然顺畅，应善用过渡词和逻辑连接词（如"然而""此外""为此""鉴于此"等）建立清晰的逻辑关系，确保整篇方法论述读起来一气呵成。整个方法部分必须以完整的学术 prose 形式呈现，严禁使用任何列点或清单格式。

本研究提出一种面向知识发现的SEKG构建框架。通过深入分析科学实验内部特征，该框架基于事件知识图谱技术，采用自顶向下和自底向上相结合的方式对科学实验进行动态、结构化、细粒度、富语义的知识表示与建模，构建SEKG模式层。在此基础上，基于人工标注数据集引入大语言模型技术，通过设计针对性的提示工程(Prompt Engineering)从科技文献中半自动标注科学实验知识与数据，构建SEKG数据层中的实例数据。SEKG构建框架分为两部分:科学实验Schema设计和科学实验知识图谱构建，如图1所示。

## 注意事项
必须严格执行分步流程（第一步画图，第二步撰写文段，第三步检查），图文务必利用代码块隔离。撰写必须使用客观、严谨、专业的学术语言，杜绝任何口语化表达。在内容组织上，前后段落之间的转折和递进必须自然流畅。最重要的是，**整个方法框架部分必须以完整的学术文段（prose）形式呈现**，每个模块的描述、每个技术细节的说明都应融入连贯的段落之中，**严禁使用任何形式的列点、清单或条目化形式**展示方法内容。若在最后的检查环节发现存在列点情况，必须自行重写为完整段落。
