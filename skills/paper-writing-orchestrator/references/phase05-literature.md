# Phase 0.5：文献广域检索与汇总

## 目标
基于 outline.md 中各章节涉及的研究主题，进行文献检索，生成一份结构化的 literature_pool.md 文献池，供后续所有章节引用。

## 前置条件
- outline.md 已存在且经用户确认

## 执行步骤

### Step 1：提取检索主题
读取 outline.md，从每个小节的注释中提取：
- **核心要点** → 主要检索关键词
- **预计引用方向** → 检索主题分类

整理出一份检索计划，格式如下：

```
检索主题列表：
1. [主题] xxx Transformer 架构改进 → 关键词: "xxx transformer", "attention mechanism improvement"
2. [主题] 两阶段训练策略 → 关键词: "two-stage training", "pretrain finetune"
3. ...
```

### Step 2：⏸️ 暂停 — 确认检索计划

将检索计划呈现给用户：
"基于大纲，我整理了以下检索主题。请确认：
1. 是否有遗漏的研究方向？
2. 是否有你已知的必须引用的论文？（请提供标题或链接）
3. 检索语言偏好？（英文为主 / 中英双语）"

如果用户提供了已知论文，直接加入文献池。

### Step 3：执行检索

根据可用工具选择检索方式：

**方式 A：使用 Zotero（如果已连接）**
```
先 tool_search 加载 zotero 工具
使用 zotero:zotero_semantic_search 或 zotero:zotero_search_items 检索用户已有的文献库
```
优先从用户已有文献库中检索，因为这些是用户已经筛选过的高质量文献。

**方式 B：使用 Web Search**
```
对每个检索主题，使用 web_search 检索
优先搜索：Google Scholar、arXiv、Semantic Scholar
关键词策略：先用英文学术关键词搜索，必要时加上年份限定
```

**方式 C：混合使用**
先从 Zotero 获取用户已有文献，再用 Web Search 补充缺失的方向。

每个主题检索 5-10 篇相关论文，总计控制在 30-60 篇。

**⚠️ 检索完成后释放 MCP 端口**：
每一轮检索完成后（不论使用 Zotero 还是 Web Search），执行以下命令释放占用的 MCP 端口，避免端口冲突影响后续操作：
```bash
powershell -Command "Stop-Process -Name node -Force" 2>&1
```

### Step 4：去重与初步筛选

对检索结果进行处理：
1. **去重**：按标题和作者去重
2. **相关性筛选**：剔除明显不相关的结果
3. **时效性排序**：优先保留近 3-5 年的工作，经典开创性工作不受时间限制
4. **质量过滤**：优先保留顶会顶刊（如 NeurIPS, ICML, CVPR, ACL, Nature, Science 等）

### Step 5：分类与标注

将筛选后的论文按主题分类，并为每篇论文标注元数据：

```markdown
# Literature Pool

## 元信息
- 生成时间：[日期]
- 基于大纲版本：outline.md
- 总计文献数：[N] 篇
- 检索工具：[Zotero / Web Search / 混合]

---

## 分类：模型架构相关

### 核心论文（必引）
- [key: vaswani2017attention] Vaswani et al., "Attention Is All You Need", NeurIPS 2017
  - 摘要：提出了 Transformer 架构，基于自注意力机制...
  - 与本文关系：本文方法的基础架构
  - 适用章节：Introduction, Related Work, Method
  - 引用场景：介绍 Transformer 背景时引用

### 相关论文
- [key: wang2024xxx] Wang et al., "xxx", ICML 2024
  - 摘要：...
  - 与本文关系：与本文方法最相似的工作，主要区别在于...
  - 适用章节：Related Work, Experiment
  - 引用场景：作为 Related Work 中的重点对比对象；Experiment 中作为 baseline

---

## 分类：训练策略相关

### 核心论文（必引）
- [key: devlin2019bert] Devlin et al., "BERT: ...", NAACL 2019
  - 摘要：...
  - 与本文关系：预训练-微调范式的代表性工作
  - 适用章节：Introduction, Related Work
  - 引用场景：说明预训练范式的有效性

### 相关论文
...

---

## 分类：评估指标与数据集
...

---

## 分类：应用场景相关
...
```

**标注字段说明**：
- `key`：引用标识符，格式为 `第一作者姓氏+年份+关键词`，全文唯一
- `摘要`：2-3 句话概括核心贡献
- `与本文关系`：说明这篇论文和用户的工作之间的具体关系
- `适用章节`：列出可能引用此论文的章节
- `引用场景`：具体在什么语境下引用此论文
- 标记为"核心论文（必引）"的是领域基础性工作或直接相关的重要工作

### Step 6：⏸️ 暂停 — 用户审阅文献池

将 literature_pool.md 呈现给用户：

"文献池初稿已完成，共 [N] 篇论文，分为 [M] 个类别。请审阅：
1. 是否有重要论文遗漏？请提供标题或链接，我来补充。
2. 是否有不相关的论文需要移除？
3. '核心论文'和'相关论文'的分级是否合理？
4. '与本文关系'的描述是否准确？
5. 分类方式是否合理？是否需要调整？"

### Step 7：迭代修改
根据用户反馈：
- 补充遗漏论文（可能需要额外检索）
- 移除不相关论文
- 调整分类和标注
- 多轮直到用户满意

### Step 8：输出 literature_pool.md
用户确认后保存到项目根目录。

提示用户：
"literature_pool.md 已生成。接下来请：
1. 开一个新对话
2. 上传项目文件（包含 outline.md 和 literature_pool.md）
3. 使用启动模板进入阶段 1（撰写 Introduction）"

## 关于 Related Work 阶段的补充检索

在阶段 2（撰写 Related Work）时，可能需要针对性补充文献。此时：
1. 在当前文献池基础上识别缺口
2. 补充检索并将新文献追加到 literature_pool.md 对应分类中
3. 新增文献标注 `<!-- ADDED_IN_PHASE_2 -->` 以区分

## 常见问题

**Q: 用户已经有 Zotero 文献库，里面有大量论文怎么办？**
A: 优先从 Zotero 库中检索和筛选，这样可以直接利用用户已有的组织结构。Web Search 只用于补充 Zotero 中没有的方向。

**Q: 检索到的论文太多怎么办？**
A: 每个分类保留 5-8 篇核心论文 + 3-5 篇补充论文。文献池总量控制在 60 篇以内，避免后续对话中上下文溢出。

**Q: 用户不确定要投哪个期刊怎么办？**
A: 文献池阶段不需要确定期刊，只需要覆盖研究主题即可。引用格式在最终转 Word 时统一处理。

**Q: 无法访问某些论文的全文怎么办？**
A: 文献池只需要标题、作者、摘要级别的信息。如果从搜索结果中只能获取有限信息，据实填写，标注 `[摘要信息有限，待用户补充]`。
