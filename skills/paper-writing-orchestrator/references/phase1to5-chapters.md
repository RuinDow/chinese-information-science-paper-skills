# Phase 1-5：逐章撰写

## 目标
每章在独立对话中完成撰写，使用对应的 generator skill，将产出填入 paper_content.md 中对应标题下方。

## 前置条件
- outline.md 已存在且经用户确认
- literature_pool.md 已存在且经用户确认
- paper_content.md 存在（阶段 0 生成的标题骨架，可能已有前几章正文内容）

## 通用执行流程（每章都遵循）

### Step 1：建立上下文

按顺序读取以下文件：

```
1. view outline.md                 ← 了解全局结构，定位当前章节的要点和注释
2. view paper_content.md           ← 了解已完成章节（如果有），保持术语和风格一致
3. view literature_pool.md         ← 获取可引用的文献及其引用场景
```

**重要**：读取 paper_content.md 时，重点关注：
- 已使用的术语和符号定义（后续章节必须保持一致）
- 已引用的文献（避免重复介绍，但可以再次引用）
- 已提及但未展开的内容（本章可能需要展开）
- 前文对本章的预告或引导句（本章开头要自然衔接）

### Step 2：加载 Generator Skill

```
view skills/[generator-skill-name]/         ← 查看目录结构
view skills/[generator-skill-name]/SKILL.md  ← 读取主指引
```

如果 skill 目录中包含 scripts/、references/ 等子目录，按 SKILL.md 中的指示调用。

**不要跳过目录查看步骤**，子 skill 可能包含：
- `scripts/` — 数据处理脚本、文献检索脚本、图表生成脚本等
- `references/` — 写作范例、风格指南、领域特定术语表等
- `assets/` — 模板文件、图片素材等

### Step 3：读取项目素材

根据 outline.md 中当前章节的注释，读取需要的项目素材：
- **关键数据引用** → 读取对应的实验数据文件
- **相关代码** → 读取对应的代码段
- **预计图表** → 确认图表文件是否存在

### Step 4：撰写本章

按照 generator skill 的指引撰写。撰写时注意：

**引用处理**：
- 从 literature_pool.md 中选择 `适用章节` 包含当前章节的文献
- 使用文献的 `key` 作为引用标识：[author2024keyword]
- 参考 `引用场景` 字段确定在什么语境下引用
- 核心论文（必引）必须出现在对应章节中

**跨章节连贯性**：
- 本章开头要自然衔接上一章的结尾
- 术语、符号、缩写要与前文保持一致
- 如果需要引用前文内容，使用 "As discussed in Section X" 等过渡
- 如果需要预告后文内容，使用 "We will elaborate on this in Section X"

**数据和图表**：
- 引用实验数据时，确保数值与 experiments/ 中的原始数据一致
- 图表引用使用 Figure X / Table X 格式
- 在正文中描述图表的关键发现，不要只放一个引用

### Step 5：⏸️ 暂停 — 用户审阅

将本章内容呈现给用户：

"[章节名] 初稿已完成（约 [N] 字/词）。请审阅：
1. 技术内容是否准确？
2. 论述逻辑是否通顺？
3. 引用是否合适？是否需要增减？
4. 是否有需要补充的内容？
5. 语言风格是否满意？

你可以提出修改意见，或者直接在文本上标注修改。"

### Step 6：迭代修改

根据用户反馈修改，使用 `str_replace` 进行精确编辑，避免重写整章。
多轮迭代直到用户满意。

### Step 7：填入 paper_content.md

用户确认后，将本章内容填入 paper_content.md 中**对应的标题下方**（不是追加到末尾）。

**操作方式**：使用 `str_replace` 将对应标题下的 `[待撰写]` 替换为实际正文内容，并将 STATUS 从 `pending` 改为 `completed`。

**内容拆分规则**：如果本章产出的内容需要分散放置到不同标题下，必须拆分后分别填入。常见情况：
- Related Work 章节：正文放在 `## 2. 相关工作` 及其子标题下，本章引用的参考文献信息追加到 `## 参考文献` 下
- Experiment 章节：正文放在 `## 4. 实验` 下，新增的图表说明可能需要放在对应子标题下
- 任何章节产生的参考文献条目，都应追加到 `## 参考文献` 下（不是放在本章末尾）

**示例操作**：
```
# 1. 替换本章正文
str_replace: [待撰写] → 实际正文（在 ## 2. 相关工作 标题下）

# 2. 更新状态
str_replace: <!-- STATUS: pending --> → <!-- STATUS: completed -->（对应章节的注释）

# 3. 追加参考文献（如有）
str_replace: [待生成] → 已有参考文献 + 本章新增参考文献（在 ## 参考文献 标题下）
```

### Step 8：提示下一步

"[章节名] 已完成并追加到 paper_content.md。接下来：
1. 开一个新对话
2. 上传更新后的项目文件（确保 paper_content.md 已更新）
3. 使用启动模板进入下一章撰写"

## 各章节特殊说明

### 对话 1：Introduction

**输入重点**：outline.md 中 Introduction 部分的注释
**写作重点**：
- 研究背景和动机（为什么这个问题重要）
- 现有方法的局限性（简要提及，详细分析留给 Related Work）
- 本文贡献总结（通常 3-4 点，bullet points）
- 论文组织结构概述（"The rest of this paper is organized as follows..."）
**引用特点**：引用量中等，主要是领域开创性工作和问题定义相关论文
**与其他章节的关系**：设定全文基调，提出的问题要在后续章节中得到回答

### 对话 2：Related Work

**输入重点**：literature_pool.md 是本章的核心输入
**写作重点**：
- 按主题或方法论组织，不要按时间线罗列
- 对每类工作要有统一的分析框架（解决什么问题、用什么方法、有什么局限）
- 明确指出现有工作与本文的区别和联系
- 最后一段总结现有工作的整体不足，自然过渡到本文方法
**引用特点**：引用量最大，几乎要用到 literature_pool.md 中的大部分文献
**特殊操作**：如果文献池中某个方向的论文不够，需要补充检索：
1. 识别缺口方向
2. 执行补充检索（web_search 或 Zotero）
3. 检索完成后释放 MCP 端口：`powershell -Command "Stop-Process -Name node -Force" 2>&1`
4. 将新文献追加到 literature_pool.md，标注 `<!-- ADDED_IN_PHASE_2 -->`
5. 继续撰写

### 对话 3：Method

**输入重点**：outline.md 中的方法要点 + experiments/ 目录中的模型代码
**写作重点**：
- 先给出方法的整体框架（overview），再逐一详述各模块
- 数学公式要完整且符号一致
- 关键设计选择要给出 motivation（为什么这样设计）
- 算法伪代码（如有）用 Algorithm 环境
**引用特点**：中等，主要引用方法借鉴的相关工作
**与其他章节的关系**：Introduction 中提到的贡献点要在这里对应展开

### 对话 4：Experiment

**输入重点**：experiments/ 目录中的数据和结果图表
**写作重点**：
- 实验设置（数据集、评估指标、baselines、实现细节）
- 主实验结果（与 baselines 的对比）
- 消融实验（验证各模块的贡献）
- 案例分析 / 可视化（如有）
- 结果讨论（为什么本方法更好 / 在什么情况下不好）
**引用特点**：引用 baseline 方法的论文、数据集论文、评估指标论文
**数据准确性**：所有数值必须与 experiments/ 中的原始数据一致，AI 不得编造实验数据
**⚠️ 关键提醒**：
- 读取实验数据时要仔细核对数值
- 如果 outline.md 中标注了 "[待补充实验数据]"，提醒用户提供
- 表格和图表的描述要与实际数据严格一致

### 对话 5：Conclusion & Abstract

**输入重点**：paper_content.md 全文（此时已有 1-4 章）
**写作重点**：

Conclusion：
- 简要回顾研究问题和方法
- 总结主要发现（呼应 Introduction 的贡献点）
- 讨论局限性（诚实且建设性）
- 未来工作方向

Abstract：
- 最后写 Abstract（因为它是对全文的高度浓缩）
- 通常 150-300 词（根据目标期刊要求调整）
- 结构：背景-问题-方法-结果-结论
- 不要引用参考文献，不要使用缩写（除非非常常见）

**引用特点**：Conclusion 中一般不引用文献；Abstract 中绝不引用
**与其他章节的关系**：这是对全文的总结，必须与前四章严格一致

## 质量检查清单（每章完成后自查）

- [ ] 术语和符号与前文一致
- [ ] 引用标识符与 literature_pool.md 中的 key 一致
- [ ] 实验数值与原始数据一致（仅 Experiment 章节）
- [ ] 章节开头与上一章结尾自然衔接
- [ ] 所有 "核心论文（必引）" 中标注了本章的论文都已引用
- [ ] 没有突兀的逻辑跳转
- [ ] 图表引用编号连续且正确
