# 相关研究部分质量检查清单

## 🔴 硬性要求检查（一票否决项）

### 文献数量要求（必须达标）
- [ ] **检索阶段**：检索获取的文献总数 **≥40篇**
- [ ] **最终引用**：综述中引用的文献总数 **≥20篇**
- [ ] **各小节分布**：每个小节至少引用 **8-12篇** 文献

**未达标处理：**
- 如检索不足40篇 → 必须扩大检索范围、调整检索词、增加数据库
- 如最终引用不足20篇 → 必须补充检索，不能降低标准

### 文献真实性检查（学术诚信底线）
- [ ] **全部真实来源**：所有引用的文献都来自实际检索结果
- [ ] **无编造文献**：没有虚构作者、标题、年份、期刊等信息
- [ ] **可追溯验证**：每篇引用的文献都能在检索日志中找到对应条目
- [ ] **无冒充文献**：没有将未检索到的文献混入引用列表

**⚠️ 严禁行为：**
- ❌ 编造不存在的文献
- ❌ 虚构文献的作者、标题、年份、期刊
- ❌ 将未检索到的文献冒充已检索到的文献
- ❌ 为了凑数而降低文献质量标准

---

## 总体检查

### 结构
- [ ] 按主题分小节（2-4个小节）
- [ ] 小节标题清晰，体现研究维度
- [ ] 标题层级一致（如"2.1"、"2.2"）
- [ ] 总字数在 1500-2500 字范围内

### 内容完整性
- [ ] 每个小节包含研究脉络概述（2-3句）
- [ ] 每个小节包含代表性工作梳理
- [ ] 每个小节末尾有小结（指出不足/空白）
- [ ] 综述末尾有逻辑闭环过渡段
- [ ] 文末有"参考文献"章节（GB/T 7714-2015格式）

### 逻辑性
- [ ] 小节之间有逻辑关联和过渡
- [ ] 每篇引用文献都说明了与本文的关系
- [ ] 综述末尾形成"前人做了什么→还缺什么→所以本文做什么"的闭环
- [ ] 不是简单的文献堆砌

---

## 文献筛选与组织检查

### 严格筛选标准
- [ ] **主题相关性**：每篇文献主题与小节标题直接相关
- [ ] **逻辑支撑性**：每篇文献能直接支撑小节核心观点
- [ ] **文献类型**：优先原创研究，慎用综述论文作为例子
- [ ] **排除不相关**：排除如"数智赋能科研创新"与"专家隐性知识形式化"这类不相关文献

### 论文归类组织
- [ ] 每个小节必须先归类再撰写（不能逐篇罗列）
- [ ] 确定分类维度（时间演进/方法论流派/问题演进/应用场景）
- [ ] 每个类别有明确名称（如"早期基于BERT的方法"）
- [ ] 每个类别内部按时间或重要性排序
- [ ] 每篇文献明确归属到某个类别

### 类别间逻辑衔接
- [ ] 类别之间必须有过渡句，说明逻辑关系
- [ ] 过渡句要指出上一类别的局限/问题，引出下一类别
- [ ] 示例格式："早期的XX方法虽然...，但其存在YY局限。为此，研究者提出了..."
- [ ] 整体形成逻辑递进（时间演进/问题解决/方法改进）

### 述评与结论紧扣
- [ ] 每个类别末尾总结该类的贡献和局限
- [ ] 所述局限必须与本文研究问题直接相关
- [ ] 为最后的逻辑闭环积累"论据"

---

## 文献引用检查

### 数量与覆盖
- [ ] 引用文献总数在 20-40 篇
- [ ] 中英文文献混合
- [ ] 近3年文献占比 >70%
- [ ] 包含该领域的经典/奠基性工作
- [ ] 各小节文献分布均衡

### 正文引用格式
- [ ] 正文引用使用上标数字，如"张三等[1]提出..."
- [ ] 多篇引用格式正确：[2,3]、[4-6]
- [ ] 引用编号连续
- [ ] 同一文献在不同位置引用编号一致
- [ ] 无遗漏引用（参考文献列表包含所有正文引用的文献）
- [ ] 无多余引用（未在正文中引用的文献不出现在参考文献列表中）

### 参考文献列表格式（GB/T 7714-2015）
- [ ] 文末有"参考文献"章节（二级标题）
- [ ] 参考文献按引用顺序编号 `[1]`, `[2]`, `[3]`...
- [ ] **期刊论文** `[J]`：作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
- [ ] **专著** `[M]`：作者. 书名[M]. 出版地: 出版者, 出版年.
- [ ] **会议论文** `[C]`：作者. 题名[C]//会议名. 出版地: 出版者, 年: 页码.
- [ ] **学位论文** `[D]`：作者. 题名[D]. 保存地: 保存单位, 年.
- [ ] **电子文献** `[EB/OL]`：作者. 题名[EB/OL]. (发布日期)[引用日期]. 网址.
- [ ] **标准** `[S]`：标准编号, 标准名称[S].
- [ ] 作者人数≤3时全部列出
- [ ] 作者人数>3时列前3位加", 等"（中文）或", et al"（英文）
- [ ] 标点符号使用正确（英文句点、逗号后加空格）
- [ ] 文献类型标识正确（[J], [M], [C], [D], [EB/OL], [S]）

### 引用质量
- [ ] 每篇引用都有实质性描述（非仅列出）
- [ ] 引用内容准确（作者、方法、结论）
- [ ] 引用来源可靠（核心期刊、高水平会议）

## 语言与风格检查

### 学术规范
- [ ] 使用学术中文，客观严谨
- [ ] 避免口语化表达
- [ ] 使用"研究表明"、"结果显示"等客观表述
- [ ] 评价措辞委婉："仍存在一定的局限性"、"有待进一步研究"

### 术语规范
- [ ] 英文缩写首次出现给出全称
- [ ] 同一概念全文使用统一术语
- [ ] 专业术语使用准确

### 过渡与连贯
- [ ] 段落之间有过渡词/句
- [ ] **小节之间有明确逻辑衔接（关键！）**
- [ ] 综述整体阅读流畅

### 小节之间的过渡检查（关键！）
- [ ] 2.1结尾（或2.2开头）有过渡段
- [ ] 过渡段包含4个要素：
  - [ ] **承接上文**：简要总结2.1的核心贡献
  - [ ] **指出关键问题**：点明2.1的局限（与2.2相关）
  - [ ] **逻辑连接**：说明为什么2.2的研究是必要的
  - [ ] **引出下文**：自然过渡到2.2的技术/方法
- [ ] 过渡逻辑示例：
  ```
  上述研究在专家隐性知识的形式化方面取得了进展...
  然而，无论是传统方法还是智能体系统，
  其所依赖的大语言模型仍存在幻觉问题...
  因此，如何抑制大模型幻觉成为关键挑战...
  ```

## 各小节撰写检查

### 每个小节的通用检查项

#### 研究脉络概述
- [ ] 开头2-3句概述该方向的整体发展脉络
- [ ] 引出下文的具体分类论述
- [ ] 点明该小节与本文研究的关系

#### 类别组织与撰写
- [ ] 每个类别有**类别名称**（如"早期基于BERT的方法"）
- [ ] 每个类别开头有1-2句**类别概述**，说明核心特征
- [ ] 类别内部按时间或重要性排序
- [ ] **不使用综述论文作为例子**（除非是奠基性综述）

#### 类别间过渡（关键！）
- [ ] 类别之间必须有**过渡句**
- [ ] 过渡句指出上一类别的**局限/问题**
- [ ] 过渡句引出下一类别的**解决方案/改进方向**
- [ ] 过渡句示例："早期的XX方法虽然...，但其存在YY局限。为此，研究者提出了..."

#### 每篇文献的述评
- [ ] 每篇文献1-2句，说明"谁做了什么，用了什么方法，得到什么结论"
- [ ] 具体描述方法细节，而非笼统说"进行了研究"
- [ ] ✅ 正确示例："陆伟等[1]从上下文学习角度设计指令策略，构建了涵盖4个能力维度的评测框架，实验表明在抽取任务上表现良好"
- [ ] ❌ 错误示例："陆伟等[1]对学术文本挖掘进行了综述"

#### 类别小结
- [ ] 每个类别末尾总结该类的**贡献**
- [ ] 指出该类别的**局限**（与本文研究问题相关）
- [ ] 为逻辑闭环积累"论据"

- [ ] 类别1：类别名称 + 概述 + 具体文献 + 过渡句
- [ ] 类别2：类别名称 + 概述 + 具体文献 + 过渡句
- [ ] 类别3（如有）：类别名称 + 概述 + 具体文献
